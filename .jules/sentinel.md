## 2024-05-02 - SSRF Protection Misconfiguration
**Vulnerability:** A `SyntaxError` (keyword argument repeated) or silent bypass in `httpx.AsyncClient` instantiation due to supplying `event_hooks` twice, which effectively neutralized the redirect validation security hook meant to prevent SSRF via malicious redirects.
**Learning:** Python dictionary literal construction and unpacking logic means duplicate keyword arguments can overwrite earlier ones or trigger syntax errors depending on how they are written. When an `httpx` client needs multiple event hooks (e.g., both local and global), they must be combined into a single list under the same dictionary key rather than supplying multiple dictionaries.
**Prevention:** Always combine multiple functions for a single HTTPX event hook into a single list (`event_hooks={"response": [hook1, hook2]}`). Verify client instantiation via unit tests to catch SyntaxErrors early.
